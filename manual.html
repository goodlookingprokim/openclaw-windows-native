Step-by-step onboarding

Windows에서 OpenClaw를 처음 실행하는 길

설치 파일 실행부터 Telegram pairing 승인까지, 참석자가 화면을 따라가며 끝낼 수 있도록 순서를 고정했습니다.

01 Prepare

설치 전 확인

Windows 10/11, 인터넷 연결, PowerShell 또는 CMD 실행 권한이 필요합니다.

설치기는 Git for Windows, Node.js LTS, pnpm을 확인하고 가능한 경우 winget으로 설치합니다.

winget install --id Git.Git -e --source winget
winget install --id OpenJS.NodeJS.LTS -e --source winget
npm install -g pnpm@10.33.2

02 Install

설치 파일 실행

A

EXE 다운로드

OpenClawWindowsNativeSetup.exe를 실행합니다.

B

SmartScreen 확인

파일 출처가 이 GitHub Pages 사이트인지 확인한 뒤 실행합니다.

C

온보딩 선택

Gateway mode는 local, bind는 loopback, port는 18789를 기본값으로 둡니다.

Install paths

설치 후 생성되는 경로

경로역할
%USERPROFILE%\openclaw-srcOpenClaw 소스 및 빌드 산출물
%USERPROFILE%\.openclaw설정, 로그, 세션, credentials
%USERPROFILE%\Desktop\OpenClaw운영용 실행기와 로그 바로가기

03 Telegram

BotFather 토큰과 pairing 승인

BOT

새 봇 만들기

Telegram에서 @BotFather에게 /newbot을 보내고 토큰을 복사합니다.

FILE

토큰 파일 등록

설치기는 토큰 원문을 로그에 출력하지 않고 로컬 token file 경로로 등록합니다.

cd $env:USERPROFILE\openclaw-src
pnpm.cmd openclaw pairing list telegram
pnpm.cmd openclaw pairing approve telegram <PAIRING_CODE>

04 Verify

설치 검증과 운영

설치 후 바탕화면 OpenClaw 폴더에서 Gateway 시작, 상태 확인, Dashboard 열기, Telegram pairing 승인을 실행할 수 있습니다.

powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1

검증은 Git, Node.js, pnpm, OpenClaw 빌드, Gateway health, Telegram channel probe를 확인합니다.

Troubleshooting

자주 생기는 문제

winget이 없을 때

Microsoft Store에서 App Installer를 업데이트하거나 Git/Node.js를 수동 설치합니다.

pnpm install 실패

네트워크, 프록시, 백신 검사 문제일 수 있습니다. 같은 PowerShell에서 다시 실행합니다.

Telegram이 응답하지 않을 때

channels status --probe와 pairing 목록을 확인한 뒤 코드를 승인합니다.
