Guided setup

설치부터 첫 Telegram 대화까지 차근차근 진행합니다.

탭으로 언어를 바꾸고, 다운로드부터 검증까지 같은 최신 흐름을 확인합니다.

한국어 mode

설치부터 첫 Telegram 대화까지

  1. 다운로드: OpenClawWindowsNativeSetup.exe를 받고 checksum을 비교합니다.
  2. 설치: Git, Node.js 22.14+, pnpm 10.33.2를 확인하고 %USERPROFILE%\openclaw-src에 OpenClaw를 준비합니다.
  3. Dry-run: Verify-OpenClawWindowsNative.ps1 -TelegramDryRunOnly로 실제 Telegram 접속 없이 redacted artifact를 만듭니다.
  4. Telegram: BotFather 토큰은 %USERPROFILE%\.openclaw\credentials\telegram-bot-token.txt에 저장하고 --token-file로 등록합니다.
  5. 검증: pairing code를 승인한 뒤 Gateway health와 channels status --probe를 확인합니다.
Get-FileHash .\OpenClawWindowsNativeSetup.exe -Algorithm SHA256
Get-Content .\checksums.sha256
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1 -TelegramDryRunOnly
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1

Dry-run은 실제 송수신 증명이 아닙니다. 실제 Telegram 대화는 사용자의 BotFather 토큰과 pairing 승인 후 확인합니다.

English mode

From setup to the first Telegram conversation

  1. Download: get the setup EXE and compare the checksum.
  2. Install: verify Git, Node.js 22.14+, pnpm 10.33.2, then prepare OpenClaw under %USERPROFILE%\openclaw-src.
  3. Dry-run: run Verify-OpenClawWindowsNative.ps1 -TelegramDryRunOnly to create a redacted artifact without Telegram network access.
  4. Telegram: store the BotFather token in a user-local token file and register it with --token-file.
  5. Verify: approve pairing, then check Gateway health and channels status --probe.
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1 -TelegramDryRunOnly
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1

Dry-run is simulated. Live send/receive remains a user-controlled credential and pairing step.
