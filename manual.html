01 다운로드

파일을 받고 지문을 확인합니다.

SHA-256은 파일의 지문입니다. 값이 같으면 공개된 파일과 같은 파일입니다.

설치 EXE 다운로드

Get-FileHash .\OpenClawWindowsNativeSetup.exe -Algorithm SHA256
Get-Content .\checksums.sha256

a1a1154f8816c3fa26100224182df9842c00777026cda1b6afff248d501781ce

02 설치

필요한 도구를 순서대로 준비합니다.

A

Git / Node.js / pnpm

설치기가 필수 명령을 확인합니다.

B

OpenClaw

%USERPROFILE%\openclaw-src에 소스를 받고 빌드합니다.

C

Gateway

loopback, token auth, port 18789를 기본으로 사용합니다.

powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Install-OpenClawWindowsNative.ps1 -RepoRef main

03 Dry-run

실제 Telegram 토큰 전에 연습합니다.

Dry-run은 실제 Telegram에 접속하지 않습니다. 민감한 값 없이 검증 결과를 만드는 예행연습입니다.

powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1 -TelegramDryRunOnly

%USERPROFILE%\.openclaw\validation\telegram-validation.dry-run.json이 생성됩니다. 이 파일은 실제 송수신 성공 증명이 아닙니다.

04 Telegram

BotFather 토큰은 로컬 파일에만 둡니다.

1. Telegram에서 `@BotFather`와 대화합니다.

2. `/newbot`을 실행합니다.

3. 봇 이름과 사용자명을 정합니다.

4. BotFather가 출력한 토큰을 복사합니다.

5. 설치기가 `Telegram bot token`을 물으면 붙여넣습니다.

설치기는 토큰을 화면이나 로그에 출력하지 않습니다. 토큰은 사용자의 로컬 파일에 저장되고 OpenClaw 설정에는 `tokenFile` 경로가 기록됩니다.

%USERPROFILE%\.openclaw\credentials\telegram-bot-token.txt

cd $env:USERPROFILE\openclaw-src
pnpm.cmd openclaw plugins enable telegram
pnpm.cmd openclaw channels add --channel telegram --token-file "$env:USERPROFILE\.openclaw\credentials\telegram-bot-token.txt"

05 Verify

pairing을 승인하고 대화를 확인합니다.

Telegram DM 기본 정책은 pairing입니다. 처음 봇에게 메시지를 보내면 pairing code가 필요할 수 있습니다.

1. Telegram에서 본인 봇에게 메시지를 보냅니다.

2. PowerShell에서 pairing 목록을 확인합니다.

cd $env:USERPROFILE\openclaw-src

pnpm.cmd openclaw pairing list telegram

3. 표시된 코드를 승인합니다.

cd $env:USERPROFILE\openclaw-src
pnpm.cmd openclaw pairing list telegram
pnpm.cmd openclaw pairing approve telegram <PAIRING_CODE>
pnpm.cmd openclaw gateway health
pnpm.cmd openclaw channels status --probe

English mode

The same path in simple English.

Download the setup EXE, verify the checksum, run the installer from native Windows, run Telegram dry-run, then enter your BotFather token only when you are ready for the real Telegram connection.

powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1 -TelegramDryRunOnly
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1

Dry-run creates a redacted artifact. It does not prove live Telegram send/receive.
