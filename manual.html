설치 가이드

설치보다 중요한 것은 다음 행동이 분명한 안내입니다.

다운로드, 체크섬, 설치, dry-run, Telegram 연결, 대화 준비 확인까지 한 화면씩 따라가세요.

설치 흐름

준비 → 실행 → 연습 → 연결 → 확인 순서입니다.

초보자도 같은 순서로 따라가도록 모든 단계를 작은 행동으로 나눴습니다. 민감한 값은 화면이나 로그에 남기지 않고, 사용자가 직접 승인한 순간에만 로컬 파일로 저장합니다.

01 다운로드

설치 파일과 지문을 먼저 확인합니다.

릴리스 페이지에서 OpenClawWindowsNativeSetup.exe를 내려받은 뒤 체크섬을 비교하세요.

Get-FileHash .\OpenClawWindowsNativeSetup.exe -Algorithm SHA256

기대 SHA-256: 5204fa42eaecce7fc40b75afa60f67c30fdbf1814848e62c70d21a51acf7d883

다운로드 페이지Checksum 확인

02 설치

Windows 네이티브 터미널에서 설치합니다.

설치기는 WSL을 사용하지 않고 Git, Node.js, pnpm, OpenClaw 저장소, Companion Gateway 준비를 순서대로 진행합니다.

.\OpenClawWindowsNativeSetup.exe
  • 기본 저장소: https://github.com/openclaw/openclaw.git
  • 기본 브랜치: main
  • 기본 소스 폴더: %USERPROFILE%\openclaw-src
  • 기본 상태 폴더: %USERPROFILE%\.openclaw
  • 기본 Gateway 포트: 18789

03 Dry-run

실제 값을 넣기 전에 흐름을 검증합니다.

dry-run은 Telegram 연결 절차를 연습하고 JSON 검증 artifact를 만듭니다. 실제 BotFather 값이나 pairing 승인은 필요하지 않습니다.

powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1 -TelegramDryRunOnly

결과는 통과, 경고, 실패, 건너뜀 상태로 정리되며 민감한 필드 이름과 값은 허용하지 않습니다.

04 Telegram

사용자가 직접 승인한 봇만 연결합니다.

BotFather에서 받은 값은 명령줄 인자로 넘기지 않습니다. 설치기는 SecureString으로 입력받고 BSTR을 해제한 뒤, OpenClaw가 읽을 수 있는 로컬 tokenFile 경로만 등록합니다.

  • 공개 저장소, GitHub Pages, 릴리스 artifact에는 값이 들어가지 않습니다.
  • pairing code와 실제 대화 검증은 사용자 환경에서만 수행합니다.
  • 문제가 있으면 dry-run 결과와 redacted validation artifact만 공유하세요.

05 검증

첫 대화 준비 상태를 확인합니다.

Gateway 상태, channel probe, pairing 승인, OpenClaw 응답 경로를 차례대로 봅니다. 실제 Telegram 송수신은 사용자가 자신의 봇과 채팅에서 최종 확인합니다.
