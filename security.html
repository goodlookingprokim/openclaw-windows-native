보안 가이드

비밀 값을 공개하지 않고 Telegram 연결 준비를 검증합니다.

체크섬, 로컬 tokenFile, dry-run artifact, 스크립트 없는 GitHub Pages로 사용자의 보안 경계를 지킵니다.

보안 원칙

공개하지 말고, 먼저 연습하고, 로컬에서만 승인합니다.

사용자가 입력하는 민감한 값은 GitHub, Pages, 릴리스, 로그에 남기지 않는 구조를 유지합니다. 설치 전에는 체크섬을, 연결 전에는 dry-run을 확인합니다.

Checksum

실행 전 SHA-256을 비교하세요.

Get-FileHash .\OpenClawWindowsNativeSetup.exe -Algorithm SHA256

기대 값: 5204fa42eaecce7fc40b75afa60f67c30fdbf1814848e62c70d21a51acf7d883

값이 다르면 실행하지 말고 새 릴리스 파일을 다시 받으세요.

로컬 경계

Telegram 값은 로컬 tokenFile에만 둡니다.

  • 명령줄 인자로 민감한 값을 넘기지 않습니다.
  • 설치기는 SecureString 입력과 메모리 해제를 사용합니다.
  • 검증 artifact에는 민감한 필드 이름도 허용하지 않습니다.
  • GitHub Pages는 정적 HTML/CSS이며 script-src 'none'을 유지합니다.
