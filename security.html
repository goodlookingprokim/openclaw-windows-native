Security posture

보안은 사이트, 다운로드, 로컬 토큰으로 나누어 확인합니다.

GitHub Pages의 한계를 인정하고, 정적 사이트에서 가능한 보호와 사용자가 직접 검증할 수 있는 절차를 제공합니다.

Website

GitHub Pages 보안 설정

HTTPS

HTTPS enforced

GitHub Pages의 HTTPS 배포를 사용하고, Pages API에서 public HTTPS 상태를 확인합니다.

CSP

Meta CSP

외부 스크립트, object embed, form submit을 차단하는 meta CSP를 모든 HTML 페이지에 적용합니다.

ZERO

No runtime JS

랜딩과 문서 페이지는 JavaScript, 외부 폰트, 추적 픽셀 없이 정적 HTML/CSS만 사용합니다.

GitHub Pages는 사용자 지정 HTTP security header를 직접 설정할 수 없습니다. 그래서 header 전용 지시어가 필요한 보호는 메타 태그와 무스크립트 정적 설계로 보완합니다.

Integrity

다운로드 파일 무결성 확인

게시된 설치 파일의 SHA-256은 다음 값이어야 합니다.

f00a55ddaa0f870d98fef4e98c82bb2cc454564b632df8d10a23ff4a599a3d0b

Get-FileHash .\OpenClawWindowsNativeSetup.exe -Algorithm SHA256
Get-Content .\checksums.sha256

checksums.sha256 열기

Secrets

토큰은 배포물에 넣지 않습니다.

Telegram bot token

사용자가 설치 중 직접 입력하고, OpenClaw 설정에는 token file 경로만 등록합니다.

Gateway token

로컬 온보딩에서 생성/선택되며 저장소, Pages, release asset에 포함하지 않습니다.

API key/OAuth

모델 provider 인증은 공식 OpenClaw 온보딩에서 사용자가 직접 선택합니다.

Local boundary

Windows PC 안에서의 기본 경계

Gateway는 loopback 127.0.0.1과 port 18789를 기본값으로 사용합니다.

민감정보 폴더는 %USERPROFILE%\.openclaw\credentials이며, 공유하거나 스크린샷에 노출하지 않습니다.
