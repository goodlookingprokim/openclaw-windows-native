Technical spec

복잡한 설치를 작은 Windows 네이티브 단계로 나눕니다.

각 단계가 무엇을 하는지, 왜 필요한지, 성공은 어떻게 확인하는지 쉽게 설명합니다.

한국어English

한국어 mode

필요한 도구를 Windows에서 순서대로 실행합니다.

부분역할
PowerShell/CMD설치 흐름을 제어합니다.
IExpress EXE여러 파일을 하나의 설치 파일처럼 묶습니다.
Node.js + pnpmOpenClaw를 설치하고 빌드합니다.
Gateway내 PC에서 OpenClaw와 UI/Telegram을 이어줍니다.
tokenFile토큰 값 대신 로컬 파일 경로를 사용합니다.

Pipeline

큰 작업을 작은 명령으로 나눕니다.

OpenClawWindowsNativeSetup.exe
  -> OpenClaw_Windows_Native_Installer.cmd
    -> Install-OpenClawWindowsNative.ps1
      1. Git / Node.js / pnpm check
      2. OpenClaw source checkout
      3. pnpm install and build
      4. Gateway setup
      5. Telegram dry-run or tokenFile
      6. verifier

Verify

무엇이 되는지 눈으로 확인합니다.

powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1 -TelegramDryRunOnly
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1

English mode

Simple native Windows pipeline

The kit uses PowerShell/CMD, Node.js, pnpm, OpenClaw Gateway, token-file Telegram setup, and a verifier. Dry-run validation creates a redacted artifact without contacting Telegram.

OpenClawWindowsNativeSetup.exe -> installer cmd -> PowerShell installer -> OpenClaw build -> Gateway -> Telegram dry-run or tokenFile -> verifier
