Technical spec

Windows native 설치 흐름과 운영 경계를 명확하게 둡니다.

WSL, Docker, Podman 없이 PowerShell, Git, Node.js, pnpm으로 OpenClaw Gateway와 Telegram channel을 준비합니다.

Stack

채택 스택

계층선택
설치 제어Windows PowerShell 5.1+
단일 EXEWindows 내장 IExpress 패키징
런타임Node.js LTS, OpenClaw 요구 최소 22.14.0
패키지 매니저pnpm 10.33.2
소스 ref기본 main, 설치 옵션 -RepoRef <tag-or-commit> 지원
Telegram 인증channels.telegram.tokenFile

Install flow

설치 파이프라인

OpenClawWindowsNativeSetup.exe
  -> OpenClaw_Windows_Native_Installer.cmd
    -> Install-OpenClawWindowsNative.ps1
      1. Git/Node.js/pnpm 확인
      2. openclaw-src clone 또는 pull
      3. RepoRef fetch 및 checkout
      4. pnpm install --frozen-lockfile
      5. pnpm build 및 ui:build
      6. openclaw onboard
      7. telegram plugin 및 tokenFile 등록
      8. gateway install/start
      9. Verify-OpenClawWindowsNative.ps1

Verification

검증 기준

네이티브 Windows

WSL_DISTRO_NAME이 없는 PowerShell/CMD 프로세스에서 실행합니다.

빌드 산출물

dist\index.js와 dist\control-ui\index.html 존재를 확인합니다.

서비스 상태

Gateway health, port 18789 listener, Telegram channel probe를 확인합니다.
