한국어 mode

최신 코드 기준 기술 흐름

설치기는 PowerShell 5.1+, CMD wrapper, IExpress 패키지, 구조화된 엔진 모듈을 사용합니다. Companion은 Tauri v2 shell로 빌드 검증되었고, 설치 경로는 WSL/Docker/Podman을 요구하지 않습니다.

OpenClawWindowsNativeSetup.exe
  -> OpenClaw_Windows_Native_Installer.cmd
    -> Install-OpenClawWindowsNative.ps1
      -> engine/OpenClawWindowsNative.Engine.psm1
      -> Git/Node.js/pnpm check
      -> OpenClaw clone/build/ui:build
      -> onboard local gateway loopback 18789 token auth
      -> Telegram tokenFile registration
      -> Gateway install/start
      -> desktop helpers
      -> verifier and JSON/redacted status

검증기는 native Windows, Node.js 버전, OpenClaw build 산출물, Gateway config, Telegram credential source, Gateway health, channel probe, port 18789 listener를 확인합니다. -TelegramDryRunOnly는 %USERPROFILE%\.openclaw\validation\telegram-validation.dry-run.json을 생성합니다.

English mode

Current technical flow

The kit uses PowerShell 5.1+, a CMD wrapper, IExpress packaging, and a structured engine module. The Companion shell is build-verified with Tauri v2, and the install path does not require WSL, Docker, or Podman.

setup EXE -> installer CMD -> PowerShell installer -> engine module -> Git/Node/pnpm checks -> OpenClaw build -> local Gateway -> tokenFile Telegram -> desktop helpers -> verifier

Verification checks native Windows, Node.js version, OpenClaw build outputs, Gateway config, Telegram credential source, Gateway health, channel probe, and port 18789. -TelegramDryRunOnly writes a redacted dry-run artifact.
