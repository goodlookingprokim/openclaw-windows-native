Technical spec

The Companion journey is backed by a simple native pipeline.

PowerShell, Git, Node.js, pnpm, OpenClaw Gateway, Control UI, and Telegram channel setup run directly on Windows without WSL, Docker, or Podman.

Stack

Chosen components

LayerSelection
Installer controlWindows PowerShell 5.1+ with a CMD wrapper.
Single-file packageWindows built-in IExpress package.
Runtime dependencyNode.js LTS; OpenClaw minimum requirement is 22.14.0.
Package managerpnpm 10.33.2.
Source refDefault main; installer supports -RepoRef <tag-or-commit>.
Telegram credentialchannels.telegram.tokenFile.

Pipeline

Install flow

OpenClawWindowsNativeSetup.exe
  -> OpenClaw_Windows_Native_Installer.cmd
    -> Install-OpenClawWindowsNative.ps1
      1. Check Git, Node.js, and pnpm
      2. Clone or update openclaw-src
      3. Fetch and checkout RepoRef
      4. Run pnpm install --frozen-lockfile
      5. Build OpenClaw and Control UI
      6. Run OpenClaw onboarding
      7. Enable Telegram plugin and tokenFile
      8. Install/start Gateway
      9. Run Verify-OpenClawWindowsNative.ps1

Runtime boundaries

Defaults are explicit and local.

GW

Gateway

OpenClaw gateway install/start uses local mode, loopback bind, token auth, and default port 18789.

UI

Control UI

The built Control UI is opened through desktop helpers so it uses the current gateway token.

TG

Telegram

Telegram pairing protects the first conversation and can be approved from CLI or desktop helper.

Verification

What the verifier proves

Verify-OpenClawWindowsNative.ps1 checks dependency availability, OpenClaw build output, config presence, Telegram channel setup, gateway health, channel probe, and whether port 18789 is listening. -TelegramDryRunOnly creates a redacted simulated artifact without live Telegram access.

powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1 -TelegramDryRunOnly
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Verify-OpenClawWindowsNative.ps1
pnpm.cmd openclaw gateway health
pnpm.cmd openclaw channels status --probe
