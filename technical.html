기술 노트

설치 경험을 검증 가능한 기술 계약으로 정리했습니다.

Node.js 기준, Gateway, dry-run artifact, CSS-only Pages, Telegram 보안 경계를 최신 흐름에 맞게 설명합니다.

기술 개요

설치 흐름을 검증 가능한 계약으로 설명합니다.

Windows 네이티브 설치, Node.js 엔진 기준, Companion Gateway, Telegram channel 검증, 정적 Pages 보안 경계를 함께 봅니다.

구성

Windows installer + OpenClaw checkout + Companion Gateway

설치기

Windows 네이티브

PowerShell/CMD 환경에서 WSL 없이 Git checkout과 dependency 준비를 수행합니다.

런타임

Node.js >=22.14.0

설치기는 Node.js 엔진 기준을 검사하고 pnpm 기반 준비를 진행합니다.

Gateway

Port 18789

Companion Gateway와 Telegram channel 검증을 로컬 상태에서 확인합니다.

검증

검증 artifact는 민감하지 않은 상태 증거만 남깁니다.

dry-run 결과는 schemaVersion, generatedAt, channel, status, mode, checks 구조로 정리됩니다. token, secret, password, pairingCode 같은 민감 필드 이름은 허용하지 않습니다.

{
  "schemaVersion": 1,
  "channel": "telegram",
  "status": "warning",
  "mode": "dry-run",
  "checks": [
    { "name": "channel-status-probe", "status": "warning" }
  ]
}

GitHub Pages

Pages는 한국어 HTML/CSS만 게시합니다.

  • 언어 전환 UI를 제거하고 한국어 단일 안내로 정리했습니다.
  • CSP는 script-src 'none'을 유지합니다.
  • Markdown 문서로 직접 보내지 않고 HTML 안내 페이지를 제공합니다.
  • open-design은 로컬에 설치해 디자인 참고용으로 활용하며, 코드나 자산은 복사하지 않습니다.
