What ships

설치 파일 하나로 시작하고, 모든 안내는 페이지 안에서 이어집니다.

참석자는 이 사이트에서 설치 파일을 받고, 같은 사이트의 온보딩 페이지에서 준비물, 설치, Telegram 연결, 검증, 문제 해결까지 순서대로 따라갈 수 있습니다.

문서 링크는 Markdown 원문으로 빠지지 않고, 읽기 쉬운 웹 페이지로 연결됩니다. 원본 Markdown 문서는 저장소 안에 보존해 유지보수와 릴리스 기록을 남깁니다.

Onboarding map

처음 설치하는 사람을 위한 4단계 흐름

01

준비물 확인

Git, Node.js LTS, pnpm, 인터넷 연결을 설치 전에 확인합니다.

 02

EXE 실행

Windows 설치기가 도구 확인, 소스 준비, OpenClaw 빌드를 진행합니다.

 03

Telegram 연결

BotFather 토큰을 로컬 token file에 등록하고 pairing을 승인합니다.

 04

검증 및 운영

Gateway health, channel probe, 바탕화면 실행기를 확인합니다.

Download center

가장 쉬운 설치 경로와 검증 경로를 분리했습니다.

권장 다운로드 OpenClawWindowsNativeSetup.exe 200 KB Windows 설치 파일
REL

릴리스 센터

EXE, PowerShell 설치기, 체크섬, 릴리스 절차를 한 곳에서 확인합니다.

 SHA

무결성 확인

다운로드한 파일의 SHA-256이 게시값과 일치하는지 확인합니다.

 FIX

문제 해결

winget, pnpm, Dashboard token, Telegram pairing 문제를 해결합니다.

Security posture

정적 사이트, 무스크립트, 무추적, 무비밀정보 원칙

WEB

Pages 보안

GitHub Pages HTTPS, meta CSP, 외부 스크립트 없는 정적 배포를 사용합니다.

 KEY

토큰 보호

Telegram/API 토큰은 배포물에 포함하지 않고 사용자 PC에만 저장됩니다.

 PC

로컬 경계

Gateway는 loopback과 사용자 프로필 경로를 기본 운영 경계로 둡니다.
